# SCL_UNSEQUENCED / SCL_REPRODUCIBLE

Макросы аннотации чистых функций.

- Заголовок: `#include <scl/utility/attribute/unsequenced.h>`
- Агрегатор: `#include <scl/utility/attribute.h>`

---

## SCL_UNSEQUENCED

Помечает функцию как не имеющую **наблюдаемых побочных эффектов** и не
читающую **никакое изменяемое глобальное или доступное через указатели
состояние**. Возвращаемое значение зависит исключительно от аргументов.
Компилятор вправе дедуплицировать, выносить за пределы цикла, переупорядочивать
или устранять вызовы.

Соответствует атрибуту C++26 `[[unsequenced]]` и расширениям GCC/Clang
`[[gnu::const]]` / `__attribute__((const))`.

**Форма — оборачивающий макрос:** в качестве аргумента передаётся весь
возвращаемый тип вместе с декларатором.

```cpp
SCL_UNSEQUENCED(int square(int x)) { return x * x; }
SCL_UNSEQUENCED(std::size_t popcount(std::uint64_t v));
```

Тело функции `{ ... }` **не** является частью аргумента макроса, поэтому
макрос корректно размещает атрибут в позицию, требуемую каждой формой:

| Компилятор / стандарт | Результат раскрытия |
|-----------------------|---------------------|
| C++26 | `int square(int x) [[unsequenced]] { ... }` |
| GCC / Clang | `[[gnu::const]] int square(int x) { ... }` |
| Более старый GCC / Clang | `__attribute__((const)) int square(int x) { ... }` |
| MSVC / неизвестный | `int square(int x) { ... }` (без аннотации) |

### Определение наличия

| Условие | Форма | Позиция атрибута |
|---------|-------|-----------------|
| `__has_cpp_attribute(unsequenced)` (C++26) | `__VA_ARGS__ [[unsequenced]]` | после списка параметров |
| `__has_cpp_attribute(gnu::const)` (GCC, Clang) | `[[gnu::const]] __VA_ARGS__` | перед возвращаемым типом |
| `__has_attribute(const)` (GCC, Clang старше) | `__attribute__((const)) __VA_ARGS__` | перед возвращаемым типом |
| Ни одно не выполнено | `__VA_ARGS__` | *(аннотация отсутствует)* |

### Ограничения

Функция, помеченная `SCL_UNSEQUENCED`, обязана:
- Не изменять никакое состояние, наблюдаемое снаружи.
- Не читать изменяемые глобальные или локальные-в-потоке переменные.
- Не читать память через аргументы-указатели (для этого используйте `SCL_REPRODUCIBLE`).
- Как правило, не вызывать функции без `SCL_UNSEQUENCED`.

### Использование

```cpp
#include <scl/utility/attribute/unsequenced.h>

SCL_UNSEQUENCED(int square(int x)) { return x * x; }

SCL_UNSEQUENCED(std::size_t popcount(std::uint64_t v));

// Объявление и определение могут оба нести аннотацию
SCL_UNSEQUENCED(double fast_rsqrt(float x));
SCL_UNSEQUENCED(double fast_rsqrt(float x)) { /* ... */ }
```

## SCL_REPRODUCIBLE

Помечает функцию как не имеющую **наблюдаемых побочных эффектов**, но
допускающую **чтение изменяемой глобальной или доступной через указатели
памяти**. Вызовы с одинаковым видимым состоянием могут быть дедуплицированы,
однако компилятор не вправе выносить их за запись в память, которую функция
может читать.

Соответствует атрибуту C++26 `[[reproducible]]` и расширениям GCC/Clang
`[[gnu::pure]]` / `__attribute__((pure))`. Слабее, чем `SCL_UNSEQUENCED`.

**Форма — оборачивающий макрос:** аналогично `SCL_UNSEQUENCED`.

### Определение наличия

| Условие | Форма | Позиция атрибута |
|---------|-------|-----------------|
| `__has_cpp_attribute(reproducible)` (C++26) | `__VA_ARGS__ [[reproducible]]` | после списка параметров |
| `__has_cpp_attribute(gnu::pure)` (GCC, Clang) | `[[gnu::pure]] __VA_ARGS__` | перед возвращаемым типом |
| `__has_attribute(pure)` (GCC, Clang старше) | `__attribute__((pure)) __VA_ARGS__` | перед возвращаемым типом |
| Ни одно не выполнено | `__VA_ARGS__` | *(аннотация отсутствует)* |

### Ограничения

Функция, помеченная `SCL_REPRODUCIBLE`, обязана:
- Не изменять никакое состояние, наблюдаемое снаружи.
- Может читать глобальные переменные, `errno` или память через указатели.
- Не вызывать функции с побочными эффектами.

### Использование

```cpp
#include <scl/utility/attribute/unsequenced.h>

SCL_REPRODUCIBLE(std::size_t strlen_safe(const char* s));

SCL_REPRODUCIBLE(bool is_sorted(const int* data, std::size_t n));
```

---

## Сравнение

| Макрос | Читает глобальную/указательную память | Переупорядочиваем | Стандарт C++26 |
|--------|---------------------------------------|-------------------|----------------|
| `SCL_UNSEQUENCED` | Нет | Да | `[[unsequenced]]` |
| `SCL_REPRODUCIBLE` | Да (только чтение) | Ограниченно | `[[reproducible]]` |

---

## Примечания

- Оба макроса подразумевают **отсутствие записи** во внешне наблюдаемое
  состояние; нарушение — неопределённое поведение.
- Не передавайте аргументы с побочными эффектами: `f(++i)` может привести к
  устранению инкремента, если `f` аннотирована.
- Каждый макрос можно переопределить до включения заголовка:
  `#define SCL_UNSEQUENCED(...) __VA_ARGS__` или
  `#define SCL_REPRODUCIBLE(...) __VA_ARGS__`.
